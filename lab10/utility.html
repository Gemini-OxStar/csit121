How is BSL Used?

Published by Timothy Kluver on November 30, 2025.

BitstringStatusListEntry

Issuers can add a credentialStatus property to enable status info for a Verifiable Credential (VC). Such properties include id, type, statusPurpose, statusList Index, statusSize, and many more. Status list entries can be used to express the purpose of a status associated with a verifiable credential by using the statusPurpose property. Using revocation or suspension as the status purpose expresses whether a VC has been revoked or suspended, respectively. Using message as the status purpose enables an issuer to define a number of custom, descriptive messages about the status of the verifiable credential.


How Bitstring Status List Credential Functions

Any credential intended to be revocable includes in its credentialStatus field a reference to the URL of theh status list (accessed with statusListCredential) and the index it occupies in the bitstring (found with statusListIndex). To check the status of a credential, one would retrieve the status list credential from the URL specified in the statusListCredential field. Next, you would download the statusListCredential resource (Ex: /credentials/status/1). Then, decode the bitstring found in the encodedList field of the status list credential. This encodesList holds a 16KB binary string that is compressed with GZIP and encoded in Base64URL. Bits that are 0 are marked as valid; those marked with 1 are to be revoked or suspended. FInaly, you can read the bit status at statusListIndex.


Use Cases

There are three different ue cases for Bitstring Status List. First, M2M Revocation. Automatic agents or machines with delegated credentials can be easily revoked when the mandate ends. Second, temprary acces. BSL is useful in platforms with time-limited passes or event-based credentials. Finally, legal compliance. Traceability can be enables in accordsnce with frameworks like AML or eIDAS 2.0 without compromising privacy.
